# LoopWise — Project Memory

## What LoopWise Is
LoopWise is a self-serve, AI-automated SaaS platform that helps small businesses re-engage their existing customer base through automated 3-channel campaigns: **email, SMS, and ringless voicemail (RVM)**. A business owner signs up, completes a short AI-guided questionnaire, and within minutes has a fully customized campaign library running automatically. No marketing experience needed. No human involvement from LoopWise after setup.

## Business Type Approach
Business type is open free-text input from the Tally onboarding form. Customers type anything — "HVAC", "food truck", "CrossFit gym", "pediatric dentist", "tattoo studio", etc. OpenAI receives the raw text and generates campaigns specific to that exact business. No predefined categories or slugs are used in campaign generation.

## Make.com Flows

### Flow 1 — New Customer Signup ✅ LIVE
Stripe checkout.session.completed → create account in Supabase → send welcome email via SendGrid

### Flow 2 — Onboarding Campaign Generation ✅ LIVE
Tally form submission → OpenAI GPT-4o (generate 3-channel campaign copy) → store in Supabase

#### Flow 2 Technical Notes
- Tally webhook fields accessed by position index in the fields array
- Field order: [0] business_name, [1] owner_name, [2] business_type (free text), [3] state, [4] city, [5] average_ticket, [6] database_size, [7] last_outreach, [8] primary_goal, [9] voicemail_pref, [10] account_id (hidden, currently null in testing)
- OpenAI receives full raw Tally fields array via toString() and parses it internally
- JSON Parse module (Module 11) parses OpenAI message.content string into structured object
- Iterator (Module 4) loops through campaigns array from Module 11
- campaign_goal values returned by OpenAI: reactivation, winback, review_request, seasonal, referral
- account_id hidden field in Tally currently returns null — needs URL parameter fix in welcome email
- Module 5 currently has account_id hardcoded to test UUID: 1c82cb81-fc39-40cb-a836-c4d73a93dc02
- Must replace hardcoded account_id with dynamic URL parameter solution before going live
- RVM voice type currently hardcoded to "tts" in Module 6 — should read from voicemail_pref field once account_id issue is resolved
- campaign_name field using campaign_goal value as name temporarily — should be human-readable name generated by OpenAI

### Flow 3 — Unified Campaign Sender ⏳ NEXT
Trigger from Supabase scheduler → send email via SendGrid + SMS via Twilio + RVM via Slybroadcast

### Flow 4 — TTS Voicemail Generation ⏳
Text input → OpenAI TTS → MP3 file → Supabase Storage → return audio URL

### Flow 5 — Opt-Out Handling ⏳
Catch unsubscribes/opt-outs from all 3 channels → write to suppressions table in Supabase
